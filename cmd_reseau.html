Commandes Réseau Essentielles

Pour Windows et Linux

1. Configuration IP

Windows

# Afficher la configuration IP
ipconfig /all

# Libérer l'adresse IP actuelle (DHCP)
ipconfig /release

# Renouveler l'adresse IP
ipconfig /renew

Linux

# Afficher la configuration IP
ip addr show

# Libérer l'adresse IP (DHCP)
sudo dhclient -r

# Renouveler l'adresse IP
sudo dhclient

2. Test de connectivité

Windows

# Tester un hôte
ping 8.8.8.8

# Vérifier le chemin vers une destination
tracert google.com

Linux

# Tester un hôte
ping -c 4 8.8.8.8

# Vérifier le chemin vers une destination
traceroute google.com

3. Test DNS

Windows

# Résolution d'un nom de domaine
nslookup openai.com

Linux

# Résolution d'un nom de domaine
dig openai.com

# Tester avec nslookup aussi possible
nslookup openai.com

4. Connexions actives et routes

Windows

# Afficher les connexions réseau
netstat -ano

# Afficher la table de routage
route print

# Afficher la table ARP
arp -a

Linux

# Connexions TCP/UDP
ss -tulnp

# Table de routage
ip route show

# Table ARP
arp -n

5. Test de services web et téléchargement

Windows

# Tester une URL avec PowerShell
Invoke-WebRequest -Uri https://openai.com

Linux

# Tester une URL
curl -I https://openai.com

# Télécharger un fichier
wget https://example.com/fichier.zip

6. Commandes de diagnostic avancé

Windows

# Vérifier le DNS utilisé
ipconfig /displaydns

# Effacer le cache DNS
ipconfig /flushdns

# Tester les ports TCP
telnet example.com 80

Linux

# Vérifier le DNS utilisé
cat /etc/resolv.conf

# Tester les ports TCP/UDP
nc -zv example.com 80
nc -zv -u example.com 53

# Afficher la table des interfaces
ifconfig -a

7. Conseils pratiques

  • Pour tester la connectivité, commencez par votre passerelle puis Internet.
  • Renouvelez votre IP si vous perdez la connexion DHCP.
  • Vérifiez les ports ouverts avec netstat/ss ou telnet/nc.
  • Utilisez les commandes DNS pour résoudre les problèmes d’accès aux sites.
