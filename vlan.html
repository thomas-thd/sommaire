Configuration des VLAN

1. Qu’est-ce qu’un VLAN ?

Un VLAN (Virtual Local Area Network) permet de segmenter un réseau en plusieurs réseaux logiques, même si les machines sont connectées aux mêmes switches physiques. Cela améliore la sécurité, la gestion du trafic et la performance.

2. Exemple d’architecture

  • VLAN 10 : Utilisateurs (192.168.10.0/24)
  • VLAN 20 : Serveurs (192.168.20.0/24)
  • VLAN 30 : Administration (192.168.30.0/24)

3. Configuration sur un switch Cisco (exemple)

Création des VLANs :

Switch> enable
Switch# configure terminal
Switch(config)# vlan 10
Switch(config-vlan)# name UTILISATEURS
Switch(config-vlan)# exit

Switch(config)# vlan 20
Switch(config-vlan)# name SERVEURS
Switch(config-vlan)# exit

Switch(config)# vlan 30
Switch(config-vlan)# name ADMIN
Switch(config-vlan)# exit

Assigner un port au VLAN :

Switch(config)# interface fastEthernet 0/1
Switch(config-if)# switchport mode access
Switch(config-if)# switchport access vlan 10

Configurer un port en trunk (pour relier un routeur ou un autre switch) :

Switch(config)# interface fastEthernet 0/24
Switch(config-if)# switchport mode trunk
Switch(config-if)# switchport trunk allowed vlan 10,20,30

4. Routage inter-VLAN

Pour que les VLAN puissent communiquer entre eux, il faut un routeur ou un switch de niveau 3. 

Router(config)# interface g0/0.10
Router(config-subif)# encapsulation dot1Q 10
Router(config-subif)# ip address 192.168.10.1 255.255.255.0

Router(config)# interface g0/0.20
Router(config-subif)# encapsulation dot1Q 20
Router(config-subif)# ip address 192.168.20.1 255.255.255.0

Router(config)# interface g0/0.30
Router(config-subif)# encapsulation dot1Q 30
Router(config-subif)# ip address 192.168.30.1 255.255.255.0

5. Vérification

  • show vlan brief → affiche les VLAN créés
  • show interfaces trunk → vérifie les trunks
  • ping entre deux hôtes → teste la connectivité
