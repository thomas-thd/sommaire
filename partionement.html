Partitionnement Linux

Disques, partitions et systèmes de fichiers

1. Définition

Le partitionnement consiste à diviser un disque physique en plusieurs parties logiques appelées partitions. Chaque partition peut contenir un système de fichiers différent.

2. Types de partitions

Type Description
PrimairePartition principale pouvant contenir un système de fichiers bootable. Max 4 par disque MBR.
ÉtenduePartition spéciale permettant de créer des partitions logiques à l’intérieur. Sert à dépasser 4 partitions.
LogiquePartitions créées à l’intérieur d’une partition étendue. Utilisées pour les systèmes et données.

3. Systèmes de fichiers courants

Système de fichiers Utilisation
ext4Système principal pour Linux
swapZone d’échange mémoire virtuelle
vfat / FAT32Compatibilité Windows/USB
ntfsDisques partagés avec Windows
xfsGrandes partitions et serveurs

4. Commandes utiles

# Lister les disques et partitions
lsblk
fdisk -l
parted -l

# Créer une partition (fdisk pour MBR, parted pour GPT)
sudo fdisk /dev/sda

# Formater une partition
sudo mkfs.ext4 /dev/sda1       # ext4
sudo mkswap /dev/sda2          # swap

# Activer la swap
sudo swapon /dev/sda2

# Monter une partition
sudo mount /dev/sda1 /mnt

# Vérifier les systèmes de fichiers
df -h
lsblk -f

5. Exemple pratique

Disque /dev/sda de 100 Go :

1. /dev/sda1 → 20 Go → ext4 → système Linux
2. /dev/sda2 → 4 Go → swap → mémoire virtuelle
3. /dev/sda3 → 76 Go → ext4 → données utilisateur

Commandes :
sudo fdisk /dev/sda
sudo mkfs.ext4 /dev/sda1
sudo mkswap /dev/sda2
sudo mkfs.ext4 /dev/sda3
sudo swapon /dev/sda2
sudo mount /dev/sda3 /mnt/data
